UEFI PXE Secure 부팅

Cloud-i DesktopLess than 1 minute참고사항UEFI PXE Secure Boot
On This Page
  • 서버 설정
  • 클라이언트 컴퓨터 설정

서버 설정

  • 서명된 파일을 다운로드하세요. snponly.efi 파일을 다운로드하고 snponly.efi 파일을 클라우드 데스크탑 서버 설치 디렉터리에 복사합니다. 예를 들어, C:\Program Files\iscsidiskServer\
  • iscsidisk 서비스를 다시 시작하고 컴퓨터의 pxe 부팅 모드를 uefi snponly로 편집합니다.

클라이언트 컴퓨터 설정

  • UEFI 펌웨어에 VMware 인증서를 등록합니다. 다운로드 및 복사 vmware_esx40.der 그리고 vmware_sb2017.der USB에 연결하고 클라이언트 컴퓨터를 BIOS로 재부팅합니다.
  • 보안 부팅을 활성화하고 OS 유형을 "Windows UEFI 모드"로 선택합니다.
  • 키 관리, DB 관리로 들어가 Centificate “vmware_esx40.der”, “vmware_sb2017.der”를 추가하세요.

!참고: VMware 인증서를 등록할 때 SignatureOwner GUID를 묻는 메시지가 나타날 수 있습니다. 보안 부팅은 사용하는 GUID에 관계없이 올바르게 작동하지만 VMware에서는 VMware를 인증서 소유자로 식별하기 위해 GUID a3d5e95b-0a8f-4753-8735-445afb708f62를 사용할 것을 권장합니다. ( https://kb.vmware.com/s/article/2148532?lang=en_US )

  • 마지막으로 BIOS를 저장하고 보안 부팅 상태가 활성화되었는지 확인합니다.
  • UEFI PXE 보안 부팅을 테스트하려면 GPT 시스템 이미지를 사용하십시오.
  • 이 경고가 표시되면 위의 단계가 올바르게 설정되었는지 확인하세요.
