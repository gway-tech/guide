PXE 네트워크 부팅 시 시스템 이미지 업데이트

  • 먼저 클라이언트 컴퓨터를 종료하고, 수퍼유저로 전환하고, 하드웨어 드라이버나 소프트웨어를 부팅 및 설치한 다음 종료하고 save write-back 명령을 사용합니다.
  • 사용자 목록에서 "후기입 저장" 명령을 클릭하면 이 인터페이스가 팝업됩니다. 해당 디스크를 선택한 후 클라이언트 시스템의 임시 데이터가 시스템 이미지에 영구적으로 기록되거나 새 복원 지점을 생성하여 다른 클라이언트 시스템이 슈퍼 사용자가 작성한 데이터를 볼 수 있도록 합니다.
  • 저장 명령을 사용한 후, 위의 단계를 반복하고 다시 저장하면 지속적인 복원 지점이 생성됩니다.

복원 지점 파일
  • 최종 연속 복원 지점 보기는 위와 같으므로 언제든지 모든 복원 지점으로 복원할 수 있습니다. 디스크 부팅 이미지만 편집하면 됩니다.

로컬 VHD 오프라인 부팅 시 시스템 이미지 업데이트

  • 먼저 클라이언트 컴퓨터에서 superuser 명령을 사용하여 슈퍼 사용자 모드로 들어가서 하드웨어 드라이버나 소프트웨어를 설치합니다.
  • 소프트웨어를 설치한 후 클라이언트 컴퓨터에서 슈퍼 유저 명령을 사용하여 종료합니다. 클라이언트 컴퓨터가 다시 시작되면 시스템에 복원 지점을 업로드하라는 메시지가 표시됩니다.
  • 업로드 후 서버 측에서 "업데이트 시작" 명령을 사용하여 최신 업로드된 복원 지점을 사용하도록 다른 클라이언트 컴퓨터를 업데이트합니다.

단일 패키지 다중 구성, 복원 지점 설명

  • 복원 지점은 부팅하려면 완전한 시스템 이미지 아래에 마운트해야 하는 불완전한 가상 디스크입니다. 여러 하드웨어 구성이 있는 머신이 있는 경우 하나의 마스터 디스크 아래에 여러 복원 지점(구성)을 생성해야 합니다. 복원 지점의 실제 크기는 설치된 드라이버 파일의 크기뿐입니다. 이렇게 하면 디스크 공간이 절약되고 서버 디스크 캐시를 보다 효율적으로 사용할 수 있습니다.
  • 구성/복원 지점 생성 방법: 먼저 사용자를 슈퍼 사용자 모드로 전환하고 드라이버를 설치한 후 시스템을 종료한 다음 save write-back 명령을 실행합니다. 후기입 데이터 저장과 동시에 복원 지점 생성
  • 구성이 2개이고 마스터 디스크가 1개 있다고 가정할 때 최종 생성되는 시스템 이미지와 복원 지점 파일은
  • 서로 다른 구성 시스템은 서로 다른 부팅 이미지를 사용하여 하나의 패키지 다중 구성을 달성합니다.

PXE 네트워크 부팅 시 데이터 디스크 업데이트

  • 기본 설정에서 "수퍼유저가 서버 디스크를 업데이트하도록 허용" 옵션을 활성화합니다.
  • 데이터 디스크를 파티션 형태로 추가합니다.
  • 클라이언트 컴퓨터를 슈퍼유저로 설정한 다음 클라이언트 컴퓨터에 게임을 설치하고 업데이트한 후 종료하고 슈퍼유저 모드를 종료하세요. 더 이상 후기입 데이터를 저장할 필요가 없습니다.
