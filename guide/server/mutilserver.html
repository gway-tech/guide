이중 서버 핫 백업 및 로드 밸런싱

  • 로드 밸런싱 수동 구현(기존 방식)

기능 소개

이중 서버 핫 백업은 동시에 온라인으로 작동하는 두 대 이상의 서버를 달성하는 것입니다. 서버 중 하나에 오류가 발생하면 다른 서버가 즉시 작업을 대신할 수 있으며 클라이언트가 작동하지 않더라도 사용 중인 클라이언트에는 영향을 미치지 않습니다. 다시 시작해야 합니다. 듀얼 서버 핫 백업은 제로 갭 연결을 달성할 수 있으며 클라이언트는 거의 변화를 느끼지 못합니다. 또한 한 서버는 기본 서버로, 다른 서버는 백업 서버로 설정할 수도 있습니다. 기본 서버는 클라이언트의 요청을 처리하는 역할을 담당하고, 백업 서버는 기본 서버의 상태를 모니터링하고 기본 서버에 장애가 발생하면 서비스를 대신하는 역할을 담당합니다. 이중 서버 핫 백업은 디스크 없는 부팅의 안정성과 가용성을 향상시키고 단일 오류 지점으로 인한 클라이언트 오류를 방지할 수 있습니다.

로드 밸런싱이란 여러 서버를 사용하여 PXE 네트워크 부팅 서비스를 제공하고, 로드 밸런싱 장치나 소프트웨어를 통해 클라이언트 요청을 분산시켜 각 서버가 비슷한 로드를 감당하도록 하는 것을 의미합니다. 로드 밸런싱은 디스크 없는 부팅의 성능과 효율성을 향상시키고, 서버 과부하로 인해 클라이언트 응답이 느려지거나 실패하는 것을 방지할 수 있습니다.

클라이언트 설치 옵션

클라이언트 설치 시 "이중 서버 핫 백업" 옵션을 선택하세요. 기타 옵션은 일반 설치와 동일합니다. Note: 클라이언트에 "이중 서버 핫 백업"을 설치하지 않아도 여러 서버의 로드 밸런싱에 영향을 주지 않습니다.!

서버측 설정

두 서버의 설정에서 동일한 클라이언트를 추가합니다(머신 이름, IP, 이미지 경로, 게임 디스크 드라이브 문자는 동일하지만 후기입 경로는 다를 수 있음). 또한 한 서버를 먼저 설정하고 "set" 접미사 구성 파일을 다른 서버에 복사할 수도 있습니다. 그런 다음 다음과 같이 설정합니다.

자동 로드 밸런싱: "다중 서버 자동 로드 밸런싱"을 확인하세요.

수동 로드 밸런싱: 각 서버 IP를 수동으로 설정 및 추가한 후 "..." 버튼을 클릭하고 각 서버 IP를 차례로 추가합니다.

여러 네트워크 카드, 여러 쓰기 저장 디스크 설정을 갖춘 단일 서버

운영 체제에서 각 네트워크 카드의 IP를 설정하면 디스크 없는 시스템이 자동으로 두 개의 네트워크 카드를 할당하여 서로 다른 클라이언트를 연결합니다. 동시에 여러 쓰기 저장 디렉터리를 설정한 후 각 디렉터리의 쓰기 저장 크기의 균형을 자동으로 조정합니다. 후기입 디스크는 별도의 디스크 하드웨어이고 NVMe 프로토콜 PCIe SSD 솔리드 스테이트 드라이브와 같이 임의 쓰기 속도가 가장 빠른 하드 디스크인 것이 좋습니다.

로드 밸런싱 수동 구현(기존 방식)

A01부터 A10까지 10대의 머신이 있다고 가정합니다. 서버 A, 서버 B 두 대가 있습니다. 서버 A는 5개의 머신 부팅을 허용하고, 서버 B는 또 다른 5개의 머신 부팅을 허용하도록 설정할 수 있습니다. 수동 로드 밸런싱을 달성합니다. 두 서버의 컴퓨터 목록은 다음과 같습니다.
