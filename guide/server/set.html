DHCP 설정

DHCP 및 TFTP 서비스 활성화: 클라이언트 PXE 네트워크 부팅에 필요한 DHCP 및 TFTP 기능을 시작할지 여부입니다. DHCP 서버: 다음 설정이 자동으로 채워지는 DHCP 서비스의 IP 주소를 지정합니다. 시작 IP 및 끝 IP: 새 클라이언트에 자동으로 할당되는 DHCP 서비스의 시작 IP 및 끝 IP 범위입니다. 서브넷 마스크, 게이트웨이, 기본 DNS 및 대체 DNS 주소: DHCP 서비스가 클라이언트에 전송하는 네트워크 설정입니다. 다중 서버 자동 로드 밸런싱: LAN에 여러 서버가 있는 경우 이 항목을 설정하십시오. 이중 서버 핫 백업 및 로드 밸런싱을 달성할 수 있습니다.

기본 설정

쓰기 저장 디렉터리: PXE 네트워크 부팅 후 클라이언트가 쓴 데이터, 즉 클라이언트가 디스크에 쓴 임시 데이터를 저장하는 데 사용됩니다. 즉, 기본적으로 서버 측 writeback 하위 디렉터리에 배치되는 소위 writeback(작업) 디렉터리입니다. 여러 쓰기 저장 디렉터리를 설정할 수 있으며 시스템은 여러 클라이언트의 쓰기 저장 파일 저장소 디렉터리의 균형을 자동으로 조정합니다. 사용자별 쓰기 저장 한도 크기: 클라이언트가 쓰는 쓰기 저장 데이터의 크기를 제한해야 하는 경우 적절한 값으로 설정하세요. 너무 작으면 클라이언트에 블루 스크린 현상이 나타납니다. 슈퍼유저가 서버 디스크를 업데이트하도록 허용: 클라이언트의 서버 디스크에 있는 파일을 업데이트해야 하는 경우 이 항목을 선택하십시오. 이 모드에서는 더 이상 쓰기 저장 데이터를 저장할 필요가 없으며 쓰기 저장 데이터가 서버 디스크에 직접 기록됩니다. 슈퍼 유저의 온라인 기간 동안에는 서버의 데이터 디스크에 접근할 수 없습니다. 시스템 이미지 업로드 디렉터리: 클라이언트가 이미지를 업로드할 때의 저장소 디렉터리로, 기본적으로 서버 측 부팅 하위 디렉터리입니다. 관리자 비밀번호: 보조 서버가 주 서버로부터 관리를 수락할 때의 인증 비밀번호입니다.

새로운 컴퓨터

새 컴퓨터 기본 설정: 새 컴퓨터의 자동 검색, 자동 로그인, DHCP 요청 및 클라이언트 연결에 대한 기본 설정입니다. 새 컴퓨터 자동 추가: 새 컴퓨터의 자동 추가를 허용할지 여부입니다. DHCP 요청, PXE 네트워크 부팅 요청, iSCSI 연결 요청 또는 클라이언트 연결 요청이 있을 때 컴퓨터가 없으면 새 컴퓨터가 자동으로 추가됩니다. 명명 규칙: DHCP를 통해 자동으로 추가되는 새 컴퓨터에 대한 명명 규칙을 설정합니다. 시스템 이미지: 자동으로 추가된 새 컴퓨터의 기본 시스템 이미지입니다. (각 이미지를 위아래로 드래그하여 표시 순서를 조정할 수 있습니다) 디스크: 자동으로 추가된 신규 사용자의 디스크 목록입니다. 운영 체제 목록 표시 시간: 컴퓨터에 여러 시스템 이미지가 있는 경우 컴퓨터가 시작될 때 시스템 선택 목록 메뉴가 표시됩니다. 여기에서 목록을 표시하는 시간을 설정할 수 있습니다. 개인 디스크 디렉터리: 공유 디렉터리 형태의 개인 디스크의 루트 디렉터리입니다. 개인 디스크 서버: 별도의 iscsi 개인 디스크 서버의 IP입니다.

새로운 컴퓨터 이름 자동 생성 규칙

새 컴퓨터 이름 지정 규칙: DHCP 요청 또는 클라이언트 연결 요청이 있는 경우 새 컴퓨터 이름 생성 규칙입니다.
