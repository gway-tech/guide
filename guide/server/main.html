전체 디스플레이

메뉴

시스템

"시스템" -> "디스크 캐시": 서버의 디스크 캐시를 시작합니다. "시스템" -> "iSCSI 시작": DHCP, TFTP, iSCSI 가상 디스크 및 클라우드 데스크탑 서비스를 시작합니다. "시스템" -> "iSCSI 중지": 모든 클라우드 데스크탑 서비스를 중지하지만 백그라운드 프로세스는 중지하지 않습니다. 새 버전의 서버를 설치해야 하는 경우 설치를 덮어쓰십시오.

"시스템" -> "보조 서버 추가": 다중 서버 관리를 위한 보조 서버를 추가합니다. "시스템" -> "보조 서버 편집": 다중 서버 관리를 위한 보조 서버를 편집합니다. "시스템" -> "보조 서버 삭제": 다중 서버 관리를 위한 보조 서버를 삭제합니다. "시스템" -> "동기화": 여러 서버 간에 설정 및 시스템 이미지를 동기화합니다.

"시스템" -> "잠금": 클라우드 데스크탑 메인 인터페이스를 잠급니다. "시스템" -> "종료": 클라우드 데스크탑 메인 인터페이스를 닫지만 클라우드 데스크탑 백그라운드 서비스는 계속 실행 중입니다.

도구

"도구" -> "디스크 추가": iSCSI 디스크와 시스템 이미지를 추가합니다. "도구" -> "디스크 편집": iSCSI 디스크 및 시스템 이미지의 속성을 편집합니다. "도구" -> "디스크 삭제": iSCSI 디스크와 시스템 이미지를 삭제합니다.

"도구" -> "컴퓨터 추가": 클라이언트 컴퓨터를 수동으로 추가합니다. "도구" -> "컴퓨터 편집": 하나 이상의 클라이언트 컴퓨터의 속성을 보거나 편집합니다. "도구" -> "컴퓨터 삭제": 선택한 하나 이상의 클라이언트 컴퓨터를 수동으로 삭제합니다. "도구" -> "수퍼유저": 사용자 쓰기 저장 데이터를 저장하는 데 사용할 수 있는 클라이언트 컴퓨터의 상태를 전환합니다.

"도구" -> "범주 추가": 컴퓨터 또는 사용자에 대한 범주를 추가합니다. "도구" -> "카테고리 편집": 컴퓨터 또는 사용자의 카테고리를 편집합니다. "도구" -> "범주 삭제": 컴퓨터 또는 사용자의 범주를 삭제합니다.

"도구" -> "설정": 서버 구성 인터페이스를 엽니다.

툴바

디스크 보기로 전환하려면 "디스크"를 클릭하세요. 컴퓨터 보기로 전환하려면 "컴퓨터"를 클릭하세요. 서버를 구성하려면 "설정"을 클릭하세요. 디스크 캐시 관리 인터페이스를 시작하려면 "캐시"를 클릭하세요. "시작"을 클릭하여 iSCSI 서비스를 시작하세요. 서비스가 시작된 후 이 버튼은 회색으로 표시됩니다. iSCSI 서비스 중지 "중지"를 클릭하면 서비스가 중지된 후 이 버튼이 회색으로 표시됩니다. "종료"를 클릭하면 클라우드 데스크탑 기본 인터페이스가 닫히지만 클라우드 데스크탑 백그라운드 서비스는 여전히 실행 중입니다. "도움말"을 클릭하여 도움말을 엽니다.

컴퓨터 목록

컴퓨터 정보: 클라이언트 시스템의 이름, IP 주소 및 MAC, 후기입 데이터 유지 여부, 이미지 업로드 허용 여부, 사용된 이미지와 디스크 조합이 포함됩니다. 읽기-쓰기 상태: 클라이언트 시스템의 온라인 시간, 현재 읽기-쓰기 속도, 총 읽기 양 및 쓰기 양이 포함됩니다.

이미지 목록

디스크 정보: 디스크 이름, 디스크 유형, 디스크 크기, 디스크 경로가 포함됩니다. 읽기 상태: 총 읽은 양, 읽기 속도가 포함됩니다.

사용자 목록

사용자 정보: 로그인 이름, 개인 디스크 이미지 파일, 용량, 마지막 로그인 IP 포함 읽기-쓰기 상태: 사용자의 온라인 시간, 현재 읽기-쓰기 속도, 총 읽기 양 및 쓰기 양을 포함합니다.

상태바

오른쪽 하단의 정보 표시줄에는 온라인 사용자 수 및 총 사용자 수, 서버 읽기-쓰기 총량 및 총 속도, 서버 온라인 시간이 표시됩니다.
