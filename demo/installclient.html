클라이언트 설치

Windows 버전

PXE 네트워크 부팅 파티션 요구 사항:

네트워크 카드 PXE가 구식 부팅을 사용하는 경우 운영 체제를 설치하기 위해 하드 디스크를 포맷하기 전에 파티션 테이블이 MBR 형식이어야 합니다. 네트워크 카드 PXE가 UEFI 부팅을 사용하는 경우 하드 디스크 파티션 테이블은 GPT 형식이어야 합니다.

BIOS의 PXE 네트워크 카드 부팅과 로컬 하드 드라이브 부팅이 일관되는 한, 예를 들어 둘 다 UEFI,
운영 체제를 설치할 때 오류가 발생하지 않습니다!

로컬 VHD 오프라인 부팅에는 파티션 요구 사항이 없습니다.

클라이언트 설치

악성 제거를 방지하기 위해 제거된 비밀번호를 설정할 수 있습니다. 클라이언트를 원격으로 제어해야 하는 경우 설치 VNC 옵션을 확인하세요.

클라이언트 시스템 구성 참고 사항:

  1. 클라이언트 컴퓨터에는 바이러스 백신 소프트웨어를 설치하지 않는 것이 좋습니다.
  2. 서버와 클라이언트 시스템의 방화벽을 끄십시오. 방화벽을 끌 수 없는 경우 서버 방화벽에서 TCP 3266 포트를 열어주세요.
  3. LAN에 클라우드 데스크탑의 내장 DHCP 또는 클라우드 데스크탑의 PXE 파일이 있는 타사 DHCP 중 하나의 DHCP 서버만 있는지 확인하십시오.
  4. 업로드하기 전에 클라이언트 시스템을 다시 시작하십시오. 클라이언트 측에 시스템을 업로드할 때 다른 애플리케이션을 닫으십시오.
  5. 운영 체제의 시스템 디스크 C 드라이브 크기를 최대한 제어하십시오. 작을수록 좋습니다.

Linux 버전

PXE 네트워크 부팅 파티션 요구 사항: Windows와 동일합니다.

로컬 VHD 오프라인 부팅 파티션 요구 사항:

파티션은 3개까지만 가능하며, 모든 운영체제는 하나의 파티션에만 설치하고 /로 마운트할 수 있습니다. 첫 번째 파티션은 Ext4 형식, 마운트 지점은 /boot(레거시 모드), Efi 형식, 마운트 지점은 /boot/efi(UEFI 모드), 두 번째 파티션은 Ext4 형식이고 마운트 지점은 / 입니다. 세 번째 파티션은 스왑 파티션이며 마운트 지점이 없습니다.

클라이언트 요구 사항 설치

필요한 구성 요소를 다운로드하고 설치하려면 클라이언트가 인터넷에 연결되어야 합니다. 먼저 네트워크 카드를 조정하고 인터넷에 연결하십시오. 네트워크 방화벽 끄기, 모든 응용 프로그램의 인터넷 연결 허용, 모든 소스의 응용 프로그램 설치 허용, 모든 응용 프로그램 실행 허용, 응용 프로그램 보호 끄기 및 기타 옵션을 포함한 보안 옵션을 끕니다.

PXE 네트워크 부팅 클라이언트 설치

  1. xxxxIscsiclient.zip을 추출합니다.

  2. 설치 스크립트 실행

sudo bash iscsi_install.sh

로컬 VHD 오프라인 부팅 클라이언트 설치

  1. xxxxVhdclient.zip을 추출합니다.
  1. 설치 스크립트 실행
sudo bash vhd_install.sh

참고: PXE 및 VHD 클라이언트를 동시에 설치하지 마십시오.

PE 도구를 사용하여 운영 체제 업로드

PE 도구를 사용하여 운영 체제를 업로드하는 경우 모든 파티션을 선택하십시오.
